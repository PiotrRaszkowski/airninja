# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

AirNinja is a KDE-Connect-style companion linking Android and macOS (MVP: SMS bridge
Android→macOS). Design specs live in [docs/protocol.md](docs/protocol.md) and
[docs/tech-stack.md](docs/tech-stack.md) — treat them as the source of truth for the wire
protocol and stack decisions.

## Toolchain

- Java is pinned to **21.0.11-zulu** via sdkman. Before any Gradle build run `sdk env` (reads
  the module's `.sdkmanrc`). For non-interactive scripts, set `JAVA_HOME` to
  `~/.sdkman/candidates/java/21.0.11-zulu` instead of sourcing sdkman (sdkman-init breaks
  under `bash script.sh`).
- Each Gradle project carries its own wrapper (protocol-core → 8.10.2, relay → 9.5.1). Use
  `./gradlew`, not a global gradle.
- macOS needs Xcode + `xcodegen` (`brew install xcodegen`); Flutter for `android/`.

## Build & test per module

| Module | Build / test |
|--------|--------------|
| `protocol-core/` (Kotlin/JVM lib) | `cd protocol-core && ./gradlew test` |
| `macos/AirNinjaCore/` (Swift pkg) | `cd macos/AirNinjaCore && swift test` |
| `macos/` (app) | `cd macos && xcodegen generate && xcodebuild -scheme AirNinja -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO build` |
| `relay/` (Spring Boot) | `cd relay && ./gradlew build` (run: `./gradlew bootRun`) |
| `android/` (Flutter) | `cd android && flutter pub get && flutter run` |

- Single Kotlin test: `./gradlew test --tests "com.airninja.protocol.framing.FrameCodecTest"`.
- Single Swift test: `swift test --filter NoiseVectorTests`.
- **Live cross-language Noise interop check:** `./scripts/interop-test.sh` (builds both cores,
  runs a real Kotlin↔Swift TCP handshake, asserts matching SAS + encrypted round-trip).

## Architecture (the big picture)

**End-to-end encryption is a hard requirement on every transport** (including the future
relay, which only ever forwards opaque ciphertext). The crypto suite is
`Noise_XX_25519_ChaChaPoly_SHA256`; MVP uses Noise **XX always** (IK reconnect deferred).

**Two independent protocol cores, kept in lockstep by a shared spec — there is no shared
binary core:**
- `protocol-core/` — Kotlin/JVM, pure logic (no Android deps), consumed by the Flutter
  Android app via platform channels. Noise = `org.signal.forks:noise-java`.
- `macos/AirNinjaCore/` — Swift/CryptoKit mirror. Noise is **hand-rolled over CryptoKit**
  (`Sources/AirNinjaCore/Noise/`), validated against the official noise-c KAT.

The two are kept byte-compatible by **`shared/conformance/`**: `vectors.json` (Device ID,
frame encoding, SAS — regenerate with `python3 shared/conformance/generate_vectors.py`) and
`noise_xx_vector.json` (the Noise KAT). Both cores have a test that reproduces these vectors.
**If you change the wire format or crypto, you must update the vectors AND keep both core test
suites and `scripts/interop-test.sh` green.**

Layering, mirrored on both sides:
- Identity: X25519 static key; `DeviceId` = `base32(SHA-256(pubkey))[:52]`.
- `FrameCodec` / `Frame`: length-prefixed CONTROL/DATA frames; a JSON `Envelope` rides inside
  CONTROL frames.
- `NoiseHandshake` + `TransportPair`/`NoiseTransport`: the XX handshake and post-handshake
  AEAD transport. `Sas` derives the 6-digit pairing code from the handshake hash.
- `SecureChannel` (`transport/`): drives the handshake and frames ciphertext over a stream
  (`InputStream`/`OutputStream` in Kotlin, the `ByteStream` protocol in Swift), using a
  2-byte big-endian length prefix per message.

`relay/` is a stateless Java 21 + Spring Boot (virtual threads) WebSocket router that carries
**zero** cryptographic responsibility — it routes ciphertext by `deviceId`.
